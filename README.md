# TopicBuilder

A Python tool for language learners that builds topical vocabulary lists from seed words. Given a handful of words that describe a topic you want to study (e.g. "economics", "finance", "monetary"), TopicBuilder uses WordNet's semantic graph to expand the concept space, then ranks every word in the lexicon by cosine similarity to identify the most relevant vocabulary for that topic. Currently built to handle input and outputs in English and Chinese but easily extendable to other languages.

## How It Works

1. **Seed → Synsets**: Your seed words are mapped to WordNet synsets (sets of synonymous words that share a meaning).
2. **Graph expansion**: A breadth-first traversal walks outward through WordNet's semantic links (hypernyms, meronyms, causes, etc.) to discover related synsets, scoring each one by how many seed synsets reach it and how far away it is.
3. **Centroid computation**: The expanded synsets' embeddings are combined into one or more "topic centroids" — points in embedding space that represent the topic.
4. **Similarity search**: Every synset in WordNet is compared to the centroids by cosine similarity, and the associated words are returned, ranked.
5. **Top-k selection**: A final budget-allocation step picks the *k* best words, distributing slots across centroids proportionally so that multi-cluster topics stay balanced.

## Data Requirements

TopicBuilder expects synset embeddings as input. WordNet lexical mappings and semlinks are built directly from NLTK WordNet/OMW.

### Where the data comes from

Lexical mappings and semantic links are loaded directly from NLTK WordNet/OMW at initialization.
Language selection uses OMW language codes such as `eng` (English), `cmn` (Mandarin Chinese), `spa` (Spanish), etc.

**Synset embeddings** were generated by feeding each synset's gloss (definition text) to OpenAI's text-embedding-3-large model, producing one vector per synset. You could substitute any embedding model as long as every synset gets a single fixed-length vector.

### Internal lexical table (`synset_words`)

| Column | Type | Description |
|--------|------|-------------|
| `synsetid` | int | WordNet synset identifier |
| `lang` | str | OMW language code (e.g. `eng`, `cmn`, `spa`) |
| `lemma` | str | Word form in that language |

A synset has multiple rows (one row per lemma per language).

### Internal semantic-link table (`semlinks`)

| Column | Type | Description |
|--------|------|-------------|
| `synset1id` | int | Source synset |
| `synset2id` | int | Target synset |
| `link` | str | Relation type (e.g. `"hypernym"`, `"hype"`, `"mero"`, `"also"`, etc.) |

This is built directly from NLTK synset relation APIs.

### `synset_embeddings` — one embedding per synset

| Column | Type | Description |
|--------|------|-------------|
| `synsetid` | int | WordNet synset identifier |
| `embedding` | np.ndarray | A 1-D float vector (e.g. 300-dim or 768-dim) |

All embeddings must share the same dimensionality and must be L2-normalized.


## Usage

```python
from topic_builder import TopicBuilder

builder = TopicBuilder(synset_embeddings, languages=["eng", "cmn"])

# Step 1 — Expand seed words through WordNet's graph
builder.find_linked_synsets(
    ['economics', 'finance', 'fiscal', 'monetary', 'optimization'],
    max_depth=2
)

# Step 2 — Compute topic centroid(s)
builder.get_centroids(strategy='weighted_mean')

# Step 3 — Find words similar to the topic
hits = builder.find_similar_words(min_similarity=0.5, language='en')

# Step 4 — Pick the top k words
builder.top_k_words(hits, 20)
```

Any seed words not found in WordNet are silently recorded in `builder.missing_seed_words` so you can check which ones were skipped.

## API Reference

### `TopicBuilder(synset_embeddings, languages=None)`

Constructor. Loads lexical mappings and semantic links from NLTK WordNet/OMW, then builds an in-memory adjacency list.

`languages` accepts OMW codes (e.g. `eng`, `cmn`, `spa`) as a string or list.
Aliases `en` and `zh` are also accepted.

---

### `find_linked_synsets(seed_words, ...)`

Map seed words to synsets and expand outward through the semantic graph.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `seed_words` | list of str | *(required)* | Words describing your topic. Matched against lexical rows in the selected seed languages. |
| `max_depth` | int | `2` | How many hops to walk from each seed synset. Set to `0` to use only the synsets directly matched by the seed words. |
| `allowed_relations` | set of str or None | `None` | If provided, only traverse edges whose `link` value is in this set. If `None`, all relations are used. |
| `max_degree` | int | `30` | Skip expanding from any synset that has more than this many outgoing edges. Prevents the traversal from blowing up through overly general concepts. |
| `decay` | float | `0.7` | Multiplicative decay per hop. A synset reached at depth *d* contributes `decay^d` to its reach score. Must be in (0, 1]. |
| `seed_languages` | str, list[str], or None | `None` | OMW language code(s) used for seed matching. Examples: `"eng"`, `"cmn"`, `["eng","spa"]`. `None` means all available languages in the lexical table. |
| `use_lemmatization` | bool | `True` | If `True`, expands English seed forms with lightweight lemmatization (WordNet morphology when available, plus simple inflection rules). If `False`, seed matching is exact (case-insensitive). |
| `return_results` | bool | `False` | If `True`, returns the `expanded_synsets` DataFrame. |

**Sets on the instance:**

- `expanded_synsets` — DataFrame with columns `synsetid`, `lexical_hits`, `reach_score`, `score`, `min_depth`, `reached_by_seeds`. Sorted by score descending.
- `selected_synsets` — the subset of `expanded_synsets` that have embeddings (inner join).
- `missing_seed_words` — list of seed words that had no match in WordNet.

**Scoring**: each synset's `score = lexical_hits + reach_score`. A synset that is directly matched by 2 seed words and also reached at depth 1 by a third seed would score `2 + 0.7 = 2.7` (with default decay).

---

### `get_centroids(strategy, params=None)`

Reduce the expanded synset embeddings down to one or more centroid vectors that represent the topic. All centroids are L2-normalized.

| Strategy | Description | `params` |
|----------|-------------|----------|
| `mean` | Unweighted mean of all expanded synset embeddings. | — |
| `weighted_mean` | Mean weighted by each synset's topic score. Good default. | — |
| `identity` | Use every expanded synset's embedding as its own centroid. Maximum granularity; can be slow for large expansions. | — |
| `anchors` | Select the top-scoring synsets as individual centroids. Useful when you want a small number of concrete reference points. | `m` (int): take top *m* synsets. `mass` (float in (0,1]): take as many synsets as needed to cover this fraction of total score. If neither is given, defaults to top 3. |
| `kmeans` | K-Means clustering with automatic *k* selection via silhouette score. Cluster centroids are score-weighted means. | `n_clusters` (int): force a specific *k*. Or use `k_min`/`k_max` (defaults 2–10) for auto-selection. `random_state` (int, default 0). |
| `hdbscan` | HDBSCAN density-based clustering. Falls back to `weighted_mean` if no clusters are found. | `min_cluster_size` (int, default 10), `min_samples` (int or None), `epsilon` (float or None), `method` (`"eom"` or `"leaf"`, default `"eom"`). |

**Sets:** `self.centroids_df` — DataFrame with `centroid_id`, `embedding`, `score`.

---

### `find_similar_words(min_similarity=0.65, ...)`

Compare every synset embedding against each centroid by cosine similarity and return words above the threshold.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `min_similarity` | float | `0.65` | Minimum cosine similarity to include a synset. |
| `scale_threshold_by_score` | bool | `False` | If `True`, lower-scoring centroids get a slightly higher threshold, controlled by `alpha`. This tightens matches for weaker clusters. |
| `alpha` | float | `0.10` | Maximum threshold increase applied to the lowest-scoring centroid when `scale_threshold_by_score=True`. |
| `language` | str or list[str] | `"both"` | Which OMW language code(s) to return (e.g. `"eng"`, `"cmn"`, `["eng","spa"]`). Legacy aliases `"en"`, `"zh"`, and `"both"` are still accepted. |
| `word_col` | str | `"word"` | Column name for the word in the output DataFrame. |

**Returns:** DataFrame with `synsetid`, `centroid_id`, `centroid_score`, `similarity`, and the word column. Each synset appears at most once (assigned to the centroid it matched best).

---

### `top_k_words(df, k, word_col="word")`

Select the best *k* words from the output of `find_similar_words`.

The word budget is allocated across centroids in proportion to their scores, so multi-centroid topics stay balanced. After the proportional allocation, any remaining slots are filled by the globally highest-similarity words not yet selected.

| Parameter | Type | Description |
|-----------|------|-------------|
| `df` | DataFrame | Output from `find_similar_words`. |
| `k` | int | Number of words to return. |
| `word_col` | str | Name of the word column (default `"word"`). |

**Returns:** DataFrame with `centroid_id`, `centroid_score`, `similarity`, and the word column, sorted by similarity descending.

## Tips

- **Start with `max_depth=2`** and `strategy='weighted_mean'`. This is a solid default for most topics. Increase depth if you get too few results; decrease it if you're drowning in noise.
- **Use `allowed_relations`** to constrain expansion. For example, restricting to hypernyms and hyponyms keeps the traversal within the same part-of-speech taxonomy.
- **Lower `min_similarity`** (e.g. 0.4–0.5) if your embeddings are lower-dimensional or your topic is broad.
- **Check `missing_seed_words`** after calling `find_linked_synsets`. Typos or words not in WordNet will show up here (for example, "djdlkj" would be flagged as missing).
- **Clustering strategies** (`kmeans`, `hdbscan`) are most useful when your seed words span distinct sub-topics. For a tight, focused topic, `weighted_mean` is usually sufficient.
